# **第八章 服务端架构与数据库设计（完整版）**

------

# **8.1 服务端总体架构**

目标：

1. **模块化**：易于维护和扩展
2. **轻量化**：适合个人开发，可先用单机/小型服务器
3. **可扩展到多人在线**：后期可接入 TCP/WebSocket
4. **数据一致性**：保证玩家属性、装备、背包、掉落等同步

------

## **8.1.1 架构分层**

```
客户端 (Unity/前端)
        │
        ▼
-----------------------
|   网关 / API 层      |  HTTP / WebSocket
-----------------------
        │
        ▼
-----------------------
|   游戏逻辑层          |  玩家操作、战斗、掉落
-----------------------
        │
        ▼
-----------------------
|   数据存储层          |  MySQL / SQLite / Redis
-----------------------
```

### **说明**

- **网关/API层**：处理请求路由、验证玩家身份
- **游戏逻辑层**：战斗逻辑、掉落生成、技能计算
- **数据存储层**：存储玩家数据、装备、背包、关卡信息、掉落表

------

# **8.2 服务端模块划分**

1. **玩家模块**
   - 注册/登录
   - 属性计算
   - 等级/经验/战力更新
2. **装备与道具模块**
   - 背包管理
   - 装备穿戴/分解/强化
   - 道具使用
3. **怪物与掉落模块**
   - 怪物生成
   - 掉落计算
   - Boss 刷新与倒计时
4. **地图与副本模块**
   - 地图加载
   - 怪物刷新点
   - 副本关卡管理
5. **战斗模块**
   - 普通攻击/技能攻击计算
   - 暴击/吸血/伤害公式
   - 战斗记录和奖励发放
6. **货币与经济模块**
   - 金币、绑定金币
   - 消耗品购买/出售
   - 背包扩展
7. **日志与监控模块**
   - 战斗日志
   - 掉落日志
   - 玩家行为记录

------

# **8.3 数据库设计**

目标：

- **简洁高效**
- **可扩展**
- **支持 JSON 存储复杂数据（装备词条、掉落表）**

------

## **8.3.1 玩家表 (player)**

| 字段        | 类型        | 描述     |
| ----------- | ----------- | -------- |
| player_id   | BIGINT      | 主键     |
| account     | VARCHAR(50) | 账号     |
| nickname    | VARCHAR(50) | 昵称     |
| level       | INT         | 玩家等级 |
| exp         | BIGINT      | 经验值   |
| hp          | INT         | 当前生命 |
| mp          | INT         | 当前魔法 |
| atk         | INT         | 攻击力   |
| def         | INT         | 防御力   |
| crit        | FLOAT       | 暴击率   |
| money       | BIGINT      | 金币     |
| bind_money  | BIGINT      | 绑定金币 |
| last_login  | DATETIME    | 最后登录 |
| create_time | DATETIME    | 创建时间 |

------

## **8.3.2 装备表 (equipment)**

| 字段              | 类型        | 描述           |
| ----------------- | ----------- | -------------- |
| equip_id          | BIGINT      | 主键           |
| player_id         | BIGINT      | 所属玩家       |
| slot              | VARCHAR(20) | 装备部位       |
| name              | VARCHAR(50) | 装备名称       |
| quality           | VARCHAR(10) | 装备品质       |
| base_atk_min      | INT         | 基础攻击最小值 |
| base_atk_max      | INT         | 基础攻击最大值 |
| base_def          | INT         | 基础防御       |
| base_hp           | INT         | 基础生命       |
| required_level    | INT         | 需求等级       |
| extra_stats       | JSON        | 词条 JSON      |
| enhancement_level | INT         | 强化等级       |
| create_time       | DATETIME    | 创建时间       |

------

## **8.3.3 背包表 (inventory)**

| 字段         | 类型        | 描述                      |
| ------------ | ----------- | ------------------------- |
| inventory_id | BIGINT      | 主键                      |
| player_id    | BIGINT      | 所属玩家                  |
| item_id      | BIGINT      | 道具ID                    |
| quantity     | INT         | 数量                      |
| item_type    | VARCHAR(20) | 装备/消耗品/材料/任务道具 |
| extra_data   | JSON        | 附加数据（装备词条等）    |

------

## **8.3.4 怪物表 (monster)**

| 字段        | 类型        | 描述          |
| ----------- | ----------- | ------------- |
| monster_id  | BIGINT      | 主键          |
| name        | VARCHAR(50) | 名称          |
| level       | INT         | 等级          |
| hp          | INT         | 生命          |
| atk         | INT         | 攻击          |
| def         | INT         | 防御          |
| exp         | INT         | 杀死获得经验  |
| drop_table  | JSON        | 掉落表        |
| spawn_point | JSON        | 刷新坐标/时间 |

------

## **8.3.5 地图表 (map)**

| 字段         | 类型        | 描述                  |
| ------------ | ----------- | --------------------- |
| map_id       | BIGINT      | 主键                  |
| name         | VARCHAR(50) | 地图名称              |
| level_range  | VARCHAR(10) | 等级范围              |
| spawn_points | JSON        | 怪物刷新点            |
| npc_points   | JSON        | NPC位置               |
| map_type     | VARCHAR(20) | 新手村/野外/精英/Boss |
| create_time  | DATETIME    | 创建时间              |

# **8.4 API 设计示例**

### **8.4.1 登录接口**

```
POST /api/login
Request: { "account": "user1", "password": "123456" }
Response: { "player_id": 10001, "token": "xxxx", "nickname": "英雄" }
```

### **8.4.2 获取背包**

```
GET /api/inventory?player_id=10001
Response:
{
  "items": [
    {"item_id":1001,"name":"烈焰剑","slot":"weapon","extra_stats":[{"type":"CritRate","value":3}]},
    {"item_id":2001,"name":"小型血瓶","quantity":10}
  ]
}
```

### **8.4.3 装备穿戴**

```
POST /api/equip
Request: {"player_id":10001, "equip_id":1001}
Response: {"status":"success","atk":50,"def":10,"hp":200}
```

### **8.4.4 刷怪掉落**

```
GET /api/monster/drop?monster_id=2001
Response:
{
  "drops":[
    {"type":"item","id":1001,"name":"烈焰剑"},
    {"type":"gold","amount":10}
  ]
}
```

# **8.5 多人在线与扩展考虑**

- **单人 MVP**：所有逻辑可在单机或轻量服务器运行
- **多人在线**：
  - 使用 WebSocket 实时推送怪物状态、掉落、战斗
  - 玩家登录验证 + token
  - 地图刷新、Boss刷新统一管理
- **可扩展模块**：
  - 邮件系统、交易行
  - 世界 Boss 公告
  - 活动副本和排行榜

------

# **8.6 第八章总结**

1. 模块化服务端设计，易于维护
2. 数据库表覆盖玩家、装备、背包、怪物、地图
3. JSON 可直接导入装备词条、掉落表、地图刷新表
4. API 可快速对接 Unity 前端
5. 单人开发可快速验证，未来可扩展多人在线
