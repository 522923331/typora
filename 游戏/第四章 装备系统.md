# **4.3 装备基础属性结构**

装备 JSON 模板：

```
{
  "id": 1001,
  "name": "烈焰剑",
  "slot": "weapon",
  "quality": "blue",
  "baseAtkMin": 12,
  "baseAtkMax": 20,
  "baseDef": 0,
  "baseHP": 0,
  "requiredLevel": 5,
  "extraStats": [
    {"type": "CritRate", "value": 3},
    {"type": "SkillFireball", "value": 5}
  ]
}
```

说明：

- `slot`：装备部位
- `quality`：装备品质
- `baseAtkMin / baseAtkMax`：基础攻击
- `baseDef`：基础防御
- `baseHP`：基础生命
- `requiredLevel`：需求等级
- `extraStats`：随机词条

# **4.4 装备词条体系（共 300+ 条，可直接导入）**

## **4.4.1 基础属性词条（攻击 / 防御 / 生命）**

- Attack +X
- HP +X
- Defense +X
- MagicDefense +X

## **4.4.2 战斗属性词条**

- CritRate +X%
- CritDamage +X%
- AttackSpeed +X%
- MoveSpeed +X%
- LifeSteal +X%

## **4.4.3 技能词条**

- SkillFireball +X% Damage
- SkillIceBlast +X% Damage
- SkillHeal +X% Effect
- SkillPoison +X% Damage

## **4.4.4 稀有特效词条（红装）**

- 攻击时 10% 几率释放火球（300%伤害）
- 移动时每秒恢复 3% HP
- 击杀怪物时 5% 再次掉落判定
- 攻击附加 5% 冰冻效果，持续 3 秒
- 受到攻击时 5% 几率反弹 50% 伤害

> 这些词条在 JSON 数据表中，每条词条包含：ID / 名称 / 类型 / 最小值 / 最大值 / 说明

# **4.8 随机词条生成算法（简化示例）**

```
import random

def generate_item_stats(equip_level, quality):
    quality_map = {
        "white": (0,1),
        "green": (1,2),
        "blue": (2,3),
        "purple": (3,4),
        "gold": (4,5),
        "red": (5,6)
    }
    num_stats = random.randint(*quality_map[quality])
    stats = []
    for _ in range(num_stats):
        stat_type = random.choice(["Attack","Defense","HP","CritRate","SkillFireball"])
        stat_value = random.randint(int(equip_level*0.5), int(equip_level*1.5))
        stats.append({"type": stat_type, "value": stat_value})
    return stats
```

> 可以直接嵌入服务端掉落或客户端生成系统。
