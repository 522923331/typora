# **第五章 背包与道具系统（完整版）**

------

# **5.1 背包总体设计**

背包系统承载玩家所有物品，包括装备、材料、消耗品、任务道具等。

### **设计目标**

- 操作简单，直观
- 容量可扩展（初始 30 格，可升级至 60）
- 与装备系统无缝衔接
- 支持批量使用、自动拾取、排序

------

## **5.1.1 背包格子设计**

| 格子类型 | 描述               | 初始数量 | 扩展上限 |
| -------- | ------------------ | -------- | -------- |
| 装备     | 武器、防具、饰品   | 10       | 20       |
| 消耗品   | 药水、卷轴         | 10       | 20       |
| 材料     | 打怪掉落、制作材料 | 5        | 15       |
| 任务道具 | 副本、剧情所需     | 5        | 10       |

> 每格可堆叠或不可堆叠取决于物品类型。

------

## **5.1.2 背包功能**

1. **排序**

   - 按类型排序（装备 → 消耗品 → 材料 → 任务道具）
   - 按品质排序（红 → 金 → 紫 → 蓝 → 绿 → 白）

2. **批量操作**

   - 批量出售、批量使用、批量分解
   - 自动拾取设置（自动拾取白装 / 绿装 / 蓝装）

3. **背包容量扩展**

   - 消耗金币 / 绑定货币扩展背包格子

   - 公式示例：

     ```
     Cost = BaseCost × 2^(扩展次数-1)
     ```

------

# **5.2 道具分类与设计**

道具主要分 4 类：装备、消耗品、材料、任务道具。

------

## **5.2.1 消耗品**

| 道具类型 | 功能          | 堆叠上限 |
| -------- | ------------- | -------- |
| 药水     | 恢复 HP / MP  | 50       |
| 卷轴     | 增益 / 技能   | 20       |
| 临时增益 | 攻击/防御加成 | 20       |

**示例 JSON**：

```
{
  "id":2001,
  "name":"小型血瓶",
  "type":"consumable",
  "effect":{"HP":100},
  "stack":50,
  "requiredLevel":1
}
```

------

## **5.2.2 材料**

- 用于强化、制作、熔炼
- 可堆叠，初始上限 50
- 示例：

```
{
  "id":3001,
  "name":"铁矿石",
  "type":"material",
  "stack":50
}
```

------

## **5.2.3 任务道具**

- 不可出售
- 不可分解
- 存在有效期或触发副本事件
- 示例：

```
{
  "id":4001,
  "name":"魔族信件",
  "type":"questItem",
  "stack":1
}
```

------

## **5.2.4 货币系统**

| 类型                    | 描述                              |
| ----------------------- | --------------------------------- |
| 金币                    | 游戏主要货币，购买道具/扩背包     |
| 绑定金币                | 限定活动获得，不能交易            |
| 元宝 / 点券（可选后期） | 充值货币，购买特效道具 / 限定装备 |

------

# **5.3 背包操作规则**

### **5.3.1 使用道具**

- 消耗品可直接右键或拖拽至快捷栏
- 技能卷轴可直接附加技能效果

### **5.3.2 装备穿戴**

- 右键装备 → 自动穿戴对应部位
- 自动替换同部位低品质装备

### **5.3.3 分解 / 熔炼**

- 背包中可分解白绿装备 → 获得材料 / 熔炼值
- 提供快捷按钮，一键分解指定品质

------

# **5.4 自动拾取系统**

- 可设置拾取规则：
  1. 白装、绿装、蓝装
  2. 金币 > X
  3. 自动拾取优先级
- 自动拾取后可直接入背包或自动出售低品质装备

------

# **5.5 背包容量与玩家成长关系**

- 初始容量 30 格（装备 10 + 消耗品 10 + 材料 5 + 任务道具 5）
- 随等级提升或消耗金币扩展
- 公式：

```
新容量 = 初始容量 + 扩展次数 × 5
扩展花费 = 100 × 2^(扩展次数-1) 金币
```

------

# **5.6 背包 UI / UX 设计**

- **UI 样式**：复古传奇风格格子
- **显示信息**：
  - 道具图标
  - 名称 + 品质颜色
  - 数量（堆叠）
- **操作**：
  - 拖拽 → 装备 / 使用 / 丢弃
  - 双击 → 使用或穿戴
  - 右键 → 分解 / 卖出
  - 快捷栏拖拽 → 技能或消耗品快捷键

------

# **5.7 背包系统与装备系统对接**

- 装备掉落 → 背包接收
- 背包操作 → 自动穿戴 → 更新角色属性
- 分解装备 → 获得材料 → 用于熔炼 / 强化

示例流程：

```
怪物掉落烈焰剑 -> 背包入库 -> 自动穿戴或存储 -> 强化/分解
```

------

# **5.8 第五章总结**

1. 背包容量灵活，可扩展
2. 四类道具清晰，操作简单
3. 自动拾取 + 批量操作减少玩家重复操作
4. 完整 JSON 模型可直接对接装备系统
5. MVP 可直接使用容量 30、自动拾取白/绿装备，玩家即可体验战斗/爆装
