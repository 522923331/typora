# **6.1 掉落系统概述**

掉落系统是玩家刷怪、刷副本、挑战 Boss 的核心奖励机制。
 设计目标：

1. **爆装爽感**：保证玩家有惊喜和收集乐趣
2. **可控数值**：掉落概率、词条数量、品质易调整
3. **适合个人开发**：数据结构简单，JSON 或 Excel 可直接导入
4. **可扩展**：未来可加套装、任务掉落、活动掉落

------

# **6.2 掉落分类**

1. **普通怪物掉落**
   - 白装、绿装、蓝装为主
   - 掉落材料、金币
   - 概率低时触发稀有装备
2. **精英怪掉落**
   - 蓝装、紫装为主
   - 材料 / 稀有掉落概率更高
   - 可带 1–3 条随机词条
3. **Boss 掉落**
   - 紫装、金装、红装概率高
   - 保底 1 件蓝装
   - 可触发稀有词条特效
4. **副本/任务掉落**
   - 固定奖励 + 随机奖励
   - 奖励可根据副本难度递增

# **6.7 掉落算法示例（简化 Python）**

```
import random

def drop_item(drop_table):
    roll = random.random()
    cumulative = 0.0
    for drop in drop_table:
        cumulative += drop['probability']
        if roll <= cumulative:
            return drop
    return None  # 未掉落

# 使用示例
monster_drops = [
    {"type":"item","id":1001,"probability":0.6},
    {"type":"item","id":1002,"probability":0.25},
    {"type":"item","id":1003,"probability":0.1},
    {"type":"gold","min":5,"max":10,"probability":1.0}
]

drop = drop_item(monster_drops)
```

# **6.8 掉落产出平衡策略**

1. **掉落总价值控制**
   - 普通怪 → 白绿为主
   - 精英怪 → 蓝紫为主
   - Boss → 紫金红为主
2. **玩家成长与掉落挂钩**
   - 怪物等级越高 → 掉落概率高 → 玩家战力增长曲线合理
3. **自动拾取优化**
   - 玩家可设置自动拾取白/绿/蓝装
   - 高级装备需手动拾取增加收集感
